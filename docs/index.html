Load the vault folder and every book inside it shows up on the left. Load a single book directly and you skip straight to its collections.

Password books

Separate folders for separate areas of your life, each book contains its own collections and its own independent encryption settings.

Collections

These are used to split a book into subcategories. A book called Personal Passwords would have collections for "Streaming Services", "Gaming", "Bank Accounts". All to give you as much control as possible.

Plain text or encrypted, per book

Storage

Each password book can be encrypted or stored as plain text, this effects every collection within it

Plain text book

Each collection is saved as its own readable .txt file inside the book's folder. You can open it in any text editor. Fast, simple, no password to remember.

Gmail (2 attributes)
    Email: name@example.com
    Password: ••••••••••

Netflix (2 attributes)
    Email: name@example.com
    Password: ••••••••••

End
These files are exactly as protected as anything else on your computer; which is to say, not very. Use an encrypted book for anything you'd be upset to have stolen.
Encrypted book

Every collection in the book gets scrambled and saved together as one file, vault.enc. Without the password, it just looks like random noise, there's nothing readable to find.

A password you create for the book
      ↓
  (Password turned into a key)
      ↓
Data inside the book → scrambled → vault.enc
Your password isn't used as the key directly, it's passed through a slow process thousands of times first, so that someone trying to guess passwords can't brute-force their way in quickly. The key only exists in memory while the book is unlocked, and relocking the book erases it.
What Password Vault Offers

Features

In depth organization

Group credentials into books, split each book into collections, and load a single book on its own when you don't need the whole vault. As much structure as you want, no more, no less.

Offline first design

Password Vault makes no network calls. Every book, collection, and entry lives on your own disk, nothing is synced, uploaded, or phoned home.

Strong encryption

Encrypted books are scrambled into one file using AES-256, a well-tested encryption standard. No readable fragments ever land on disk.

Password generation

Generate strong, random passwords on the spot, with control over length and character types, so you're never stuck reusing the same password twice.

Search

Filter within a single collection or across the whole open book, instantly.

Copy to clipboard

One click copies any field. Sensitive values like passwords, tokens, and PINs stay masked until you ask to see them.

Important to know

Security notes

  • Fully offline. The app makes no network calls. Nothing about your vault ever leaves your machine.
  • Memory-only keys. Decryption keys exist only for the length of a session and are wiped the moment a book is relocked.
  • Plain text means plain text. Unencrypted books rely entirely on your filesystem permissions for access. Use encrypted books for anything sensitive.
  • Clipboard isn't auto-cleared. Copied values stay on your clipboard until you clear it or copy something else.
  • Not independently audited. This is a personal tool, built and shared in the open; not a commercial product with a security team behind it.
